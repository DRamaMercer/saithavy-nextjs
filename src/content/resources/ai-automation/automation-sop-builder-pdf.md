# Automation SOP Builder

## Document Standard Operating Procedures for Repeatable Automation

---

## Introduction

Automation without documentation is fragile. When the creator leaves, the automation breaks. The **Automation SOP Builder** helps you create comprehensive documentation for every automation—ensuring reliability, maintainability, and team alignment.

**What you'll get:**
- Complete SOP template for any automation
- Step-by-step documentation framework
- Diagram templates for visual workflows
- Testing and maintenance checklists
- Handoff documentation for team transitions

**Use this for:** Every automation you build, from simple Zapier zaps to complex workflow systems.

---

## Part 1: The Automation SOP Template

### SOP Document Structure

Every automation SOP should include these sections:


## Part 7: Best Practices

### Documentation Best Practices

- **Write for someone else:** Assume the reader has no context
- **Keep it current:** Update docs before, not after, changes
- **Use visuals:** Diagrams clarify complex flows
- **Version control:** Track changes and why they were made
- **Be specific:** "Set up integration" → "Connect HubSpot to Zapier using API key XYZ"
- **Include examples:** Show, don't just tell
- **Document decisions:** Why was this approach chosen?
- **Plan for failure:** Document error states explicitly

### When to Update Documentation

- [ ] Before making changes (add "proposed change" section)
- [ ] After successful deployment
- [ ] When errors occur and are resolved
- [ ] Quarterly (review all SOPs)
- [ ] When ownership changes
- [ ] When connected apps update
