## Template Overview {#overview}

### Purpose

The Weekly Sprint Planning Template provides remote teams with a structured framework to:
- Align on weekly priorities
- Assign clear ownership
- Track progress asynchronously
- Review outcomes and learn
- Maintain momentum across distributed locations

### Sprint Structure

```
┌─────────────────────────────────────────────────────────────┐
│                     WEEKLY SPRINT                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  MONDAY                       FRIDAY                        │
│  ┌─────────────┐              ┌─────────────┐              │
│  │ SPRINT      │              │ SPRINT      │              │
│  │ PLANNING    │──────────────│  REVIEW     │              │
│  │ (60-90 min) │              │ (30-60 min) │              │
│  └─────────────┘              └─────────────┘              │
│         │                             │                     │
│         v                             v                     │
│  ┌─────────────────────────────────────────────────┐       │
│  │            DAILY STANDUPS (async/sync)          │       │
│  │  Monday │ Tuesday │ Wed │ Thu │ Fri │ Wrap     │       │
│  └─────────────────────────────────────────────────┘       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### Roles

| Role | Responsibilities | Ideal For |
|------|------------------|-----------|
| **Sprint Lead** | Facilitates planning, keeps sprint on track | Rotating weekly |
| **Scribe** | Documents decisions, updates sprint board | Rotating weekly |
| **Owner** | Responsible for specific deliverable | Task-specific |
| **Reviewer** | Reviews completed work | Assigned to owner |

## Remote Sprint Modifications {#remote-modifications}

### Pre-Sprint Preparation

**Sprint Lead Checklist (Before Monday):**
- [ ] Update sprint board with carry-forward tasks
- [ ] Review upcoming deadlines and dependencies
- [ ] Identify potential conflicts (OOO, competing priorities)
- [ ] Prepare agenda for planning session
- [ ] Share pre-reading material (if async planning)

**Team Member Preparation:**
- [ ] Review task status before planning
- [ ] Come with availability constraints
- [ ] Identify any blockers from previous week
- [ ] Consider capacity for the week

### Remote Meeting Best Practices

**Planning Session Setup:**
- [ ] Video on (optional but encouraged)
- [ ] Shared document visible to all
- [ ] Timer for each section
- [ ] Break scheduled (if meeting >60 min)
- [ ] Backup async option for absent members

**Facilitation Tips:**
- Call on people explicitly (no "any thoughts?")
- Use polls for quick decisions
- Document in real-time
- Repeat decisions for confirmation
- Summarize before moving to next topic

## Async Sprint Variations {#async-variation}

### Fully Async Sprint Planning

**Timeline:**

**Friday 5 PM:** Sprint planning template shared
**Monday 10 AM:** Contributions due
**Monday 2 PM:** Sprint Lead synthesizes and posts final plan
**Monday 4 PM:** Objections/concerns deadline
**Tuesday 9 AM:** Sprint begins (with final plan confirmed)

### Async Planning Template

```markdown
# Weekly Sprint Planning - Week of [Date]

## Context
[Summary of previous week outcomes]

## Proposed Sprint Goals
[Draft 2-3 goals for team consideration]

## Proposed Tasks
[List of proposed tasks with rough estimates]

## Input Requested
Please contribute by Monday 10 AM:
1. Any changes to proposed goals
2. Tasks you want to take ownership of
3. Availability constraints this week
4. Dependencies/risks you're aware of

## Planning Doc
[Link to collaborative planning document]

## Final Plan
Will be posted Monday 2 PM after all input considered.
```

### Async Daily Standup

**Option 1: Channel Post**
- Post in #sprint-standups by end of day
- Use consistent template
- React with emoji to acknowledge review

**Option 2: Form/System**
- Fill in shared form
- Dashboard aggregates for team view
- Notifications only on blockers

**Option 3: Bot-Automated**
- Bot prompts each person at scheduled time
- DM responses posted to channel
- Summarizes blockers automatically

## Sprint Metrics & Tracking {#sprint-metrics}

### Core Sprint Metrics

**Completion Metrics:**
| Metric | Formula | Target |
|--------|---------|--------|
| Sprint Goal Success Rate | Goals achieved ÷ Goals set | ≥80% |
| Task Completion Rate | Tasks done ÷ Tasks committed | ≥85% |
| On-Time Delivery | Tasks on time ÷ Tasks done | ≥90% |
| Carry-Over Rate | Tasks moved ÷ Tasks total | ≤15% |

**Quality Metrics:**
| Metric | Formula | Target |
|--------|---------|--------|
| Rework Required | Tasks redone ÷ Tasks done | ≤5% |
| Blocker Resolution | Blockers resolved ÷ Total | ≥90% |
| Sprint Goal Clarity | Team understands goal ÷ Team size | 100% |

**Team Health Metrics:**
| Metric | Formula | Target |
|--------|---------|--------|
| Participation | Members updating status ÷ Total | 100% |
| Satisfaction | Avg satisfaction score | ≥4/5 |
| Stress Level | Avg stress score | ≤3/5 |

## Common Issues & Solutions {#common-issues}

### Issue 1: Overcommitment

**Symptom:** Consistently carrying over tasks

**Solutions:**
- Reduce tasks by 20% for next sprint
- Improve time estimates (track actual vs. estimated)
- Say no to mid-sprint additions
- Include buffer time in planning

### Issue 2: Unclear Goals

**Symptom:** Confusion about sprint priorities

**Solutions:**
- Use the "One Thing" framework
- Ensure goals are SMART
- Post goals visibly (Slack status, dashboard)
- Reference goals in daily standups

### Issue 3: Lack of Ownership

**Symptom:** Tasks without clear owners

**Solutions:**
- Every task must have ONE owner
- Assign during planning, not after
- Owners must accept task assignment
- Celebrate ownership completion publicly

### Issue 4: Zombie Sprints

**Symptom:** Going through motions, no energy

**Solutions:**
- Change format (async to sync or vice versa)
- Rotate sprint lead
- Add a fun element (theme, competition)
- Have a retrospective on the sprint process itself
- Take a sprint break (planning week only)

### Issue 5: Time Zone Challenges

**Symptom:** Some members can't attend planning/review

**Solutions:**
- Rotate meeting times (share the pain)
- Async-first planning with async review
- Recording for unavailable members
- Buddy system for updates
