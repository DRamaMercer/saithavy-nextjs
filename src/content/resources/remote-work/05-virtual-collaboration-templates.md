# Virtual Collaboration Templates

## Executive Summary

Effective virtual collaboration requires intentionality. Without the luxury of physical proximity, remote teams must use clear, structured communication to stay aligned and productive.

**This guide provides:**
- Ready-to-use templates for all common scenarios
- Best practices for virtual meetings
- Frameworks for collaborative decision-making
- Tools and techniques for async collaboration

**The result:** Less confusion, faster decisions, stronger alignment.

---

## 📋 Meeting Templates

### Pre-Meeting Checklist

**Before you schedule any meeting, ask:**
- [ ] Could this be an email, document, or recorded video?
- [ ] Is there a clear goal and agenda?
- [ ] Are only essential people invited?
- [ ] Is the time respectful of all time zones?
- [ ] Is there pre-work to prepare?

**If you answer NO to any of these, reconsider the meeting.**

## 📊 Document Collaboration Templates

### Document Collaboration Best Practices

**For Google Docs, Notion, Confluence, etc.:**

**1. Use comments, not suggestions, for discussions**
- Suggestion mode is for edits
- Comments are for questions, feedback, discussion
- Keep threads focused (one topic per comment)

**2. Be specific in your feedback**
- Bad: "This doesn't make sense"
- Good: "I'm confused about how this connects to the previous paragraph. Can you clarify?"

**3. Resolve comments when addressed**
- Don't leave threads hanging
- Mark as resolved when done
- Reply if you're NOT making the change (explain why)

**4. Use structure and formatting**
- Headings, bullets, numbered lists
- Makes documents scannable
- Use templates for consistency

**5. Version control**
- Major changes? Create new version
- Keep old versions in case you need to revert
- Document what changed in each version

**Before requesting review:**
- [ ] Document is complete (not half-done)
- [ ] Proofread for typos and grammar
- [ ] Clear structure and formatting
- [ ] Context provided (what this is, why it matters)
- [ ] Clear ask (what type of feedback needed)
- [ ] Deadline specified

**During review:**
- [ ] Provide feedback by deadline
- [ ] Be constructive and specific
- [ ] Explain your reasoning
- [ ] Differentiate between "must fix" and "nice to have"

**After review:**
- [ ] Acknowledge all feedback
- [ ] Explain why you're NOT making certain changes
- [ ] Share updated version by promised date
- [ ] Thank reviewers for their time
