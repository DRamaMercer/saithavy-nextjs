# Swarm Agency Template System

*A comprehensive, standardized template system for content creation, metadata management, intelligence reporting, and performance analytics.*

---

## System Overview

This template system provides standardized structures for the entire Swarm Agency content workflow:

```
swarm-agency/
├── README.md (this file)
├── platform-content-templates/     # Platform-specific content templates
├── metadata-templates/              # YAML front matter and metadata tracking
├── intelligence-templates/          # Weekly intelligence and competitive analysis
└── performance-templates/           # Platform health and performance dashboards
```

## Quick Start

### 1. Platform Content Templates

- Facebook

**Each Template Includes**:

## Best Practices

### 1. Use Templates Consistently

- Apply templates to every piece of content
- Use same field names across all content
- Follow naming conventions (content IDs, platform names)
- Maintain consistent formatting

### 2. Keep Metadata Accurate

- Use real dates and metrics (not placeholders)
- Document actual team members involved
- Track actual tools used (not aspirational)
- Update fields as content evolves

### 3. Track Performance Relentlessly

- Record baseline metrics before publishing
- Update actual performance after publishing
- Note what worked and what didn't
- Store iteration notes for future reference

### 4. Build Relational Graphs

- Link related content
- Track repurposing lineage
- Document A/B test results
- Map content series and campaigns

### 5. Distribute Intelligence

- Share weekly briefs with relevant teams
- Use dashboards for stakeholder reporting
- Archive reports for historical analysis
- Create actionable summaries

## Naming Conventions

### Content IDs

Format: `{platform}-{type}-{yyyy}-{nnn}`

Examples:
- `tiktok-video-2025-001`
- `instagram-carousel-2025-001`
- `youtube-long-2025-001`

### Platform Names

Use lowercase, hyphen-separated:
- `tiktok`
- `instagram`
- `youtube`
- `twitter-x`
- `linkedin`
- `pinterest`
- `facebook`
- `blog`
- `podcast`
- `email`

### Content Types

- `video`
- `image`
- `text`
- `carousel`
- `thread`
- `story`
- `reel`
- `short`
- `post`
- `article`
- `episode`
- `newsletter`

## System Architecture

### Template Relationships

```
Content Creation
    ↓
Metadata (Content DNA)
    ↓
Publish & Track
    ↓
Performance Review
    ↓
Intelligence Brief
    ↓
Strategy Adjustment
    ↓
Content Creation (Cycle repeats)
```

### Data Flow

1. **Planning**: Use intelligence brief to inform content calendar
2. **Creation**: Apply platform templates with metadata
3. **Publishing**: Track with metadata front matter
4. **Performance**: Review with performance templates
5. **Intelligence**: Aggregate findings in weekly brief
6. **Optimization**: Adjust strategy based on insights
